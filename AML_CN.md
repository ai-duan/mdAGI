# AML规范提案：智能体标记语言

## 1. 概述
AML（Agent Markup Language）是一种用于形式化描述智能体实体的标记语言规范。它通过五个标准化的维度，为智能体提供完整且最小的状态描述框架。

## 2. 核心概念

### 2.1 智能体五维模型
智能体的完整状态可被且仅需以下五个维度描述：
- **agent**：身份标识与角色定义
- **knowledge**：知识体系与认知规则
- **memory**：状态保持与经验记录
- **code**：行为能力与操作方法
- **todo**：目标队列与任务规划

### 2.2 文档即实体原则
AML文档本身即是智能体的完整状态表示。文档的加载、解析、修改分别对应智能体的实例化、状态读取和进化。

## 3. 理论基础

### 3.1 形式化定义
令 AML 为五元组：
```
AML = ⟨A, K, M, C, T⟩
其中：
A ∈ Strings  （身份描述）
K ∈ KnowledgeBase（知识集合）
M ∈ MemorySystem（记忆配置）
C ∈ CapabilitySet（能力集合）
T ∈ TaskQueue（任务队列）
```

### 3.2 合成代数
定义合成操作 ⊕：
```
AML₁ ⊕ AML₂ → AML₃
```
满足：
1. 保持五维结构
2. 可选择性地继承父代特征
3. 可能产生涌现特性

## 4. 规范特性

### 4.1 最小完备性
五个维度构成描述智能体的最小完备集，减少任何一维都将导致状态描述不完整。

### 4.2 正交独立性
各维度在语义和功能上相互独立，可单独修改而不影响其他维度。

### 4.3 可组合性
AML实体支持形式化的合成操作，能够通过结构化组合产生新实体。

### 4.4 自描述性
AML文档包含其自身的状态和行为描述，支持自检和自修改。

## 5. 标准化目标

### 5.1 互操作性
为不同平台和系统的智能体提供统一描述格式，支持跨平台协作。

### 5.2 可移植性
纯文本格式确保AML文档的平台无关性和工具链兼容性。

### 5.3 可扩展性
在保持核心五维模型不变的前提下，支持各维度的内部扩展。

## 6. 理论贡献

1. **统一描述框架**：为各类智能体提供标准化状态表示
2. **形式化基础**：建立智能体合成的代数理论基础
3. **实践指导**：为智能体系统的设计与实现提供参考规范

## 7. 适用范围
本规范适用于基于语言模型的智能体系统、任务导向的自动化实体以及需要标准化描述的智能代理。
