# Genesis Prime: AI代理运行时系统

## 概述
Genesis Prime 是一个实验性的AI代理运行时系统，旨在创建、管理和进化自主AI代理。基于Python构建，使代理能够感知环境、规划行动、执行任务并从经验中学习。

## 核心概念
- **代理创建**：通过有丝分裂产生具有专门目标的AI代理
- **任务管理**：通过分解和执行处理复杂任务
- **记忆系统**：代理通过经验学习和保留知识
- **自我进化**：代理可以合并回父级，共享获得的知识

## 主要特性
- 自主代理生命周期管理（创建、执行、合并）
- 基于Markdown格式的持久化状态存储
- 记忆提炼将经验转化为可操作的知识
- 用于代理交互的命令行界面
- 支持多种工具（文件操作、代理创建等）

## 项目结构
- `cli.py` - 与代理交互的命令行界面
- `runtime/` - 核心运行时引擎
  - `runtime.py` - 主代理运行时逻辑
  - `state.py` - 代理状态管理
  - `parser.py` - Markdown文件解析和序列化
  - `llm.py` - 简化的LLM接口
  - `executor.py` - 工具执行引擎
- `genesis_v1.md` - 初始代理DNA模板

## 快速开始

1. 确保已安装Python 3.7+
2. 克隆仓库
3. 使用默认DNA文件运行代理：
   ```bash
   python cli.py
   ```
4. 持续运行模式：
   ```bash
   python cli.py --forever
   ```
5. 运行指定周期数：
   ```bash
   python cli.py --loop 5
   ```

## 使用示例

### 持续运行


### 自定义DNA文件
```bash

# 使用自定义代理DNA文件运行
```

## 代理生命周期

1. **创建**：从Markdown格式的DNA模板创建代理
2. **感知**：代理分析环境和待处理任务
3. **规划**：代理使用内置推理制定计划
4. **行动**：代理执行工具或创建子代理
5. **记忆**：将行动和结果存储为经验
6. **提炼**：将多余的记忆转化为永久知识
7. **合并**：子代理可以合并回父级，与父级共享知识

## 可用工具

- `read_file` - 读取文件内容
- `write_file` - 将内容写入文件
- `add_task` - 向队列添加新任务
- `mitosis` - 创建专门的子代理
- `merge_child` - 将子代理合并回父级
